Case Study

A User Experience for a Portfolio & Website

Portfolio UX/UI seeks minute optimizations

By FRANCIS CACCAVALE

An optimization-seeking design approaching a portfolio/website in conjunction with responsive design while attributing to some identity that a user can relate to.

Narrowing optimizations

A primary target in this design was to seek small yet poignant optimizations, that will help a relatively trivial website/portfolio convey information at lightning speed yet retain aesthetics.

A portfolio/website user-interface and user-experience has seen numerous iterations, maybe tens of thousands. A new threshhold begins as to whether an individual wants to generate a generic 'Squarespace et cetera' template or try a hand at html/css into the influence of Squarespace templates. The only issue is that after Squarespace generates hundreds of portfolio templates for individuals globally, the landscape may become complacent as to 'this is what a portfolio has to be, UI, fonts, colors et cetera'. This design intends to create a stochastic approach to break a mold, which means to create a baseline prototype solely from experience and build a design process from a prototype.

After creating an initial low-fidelity prototype based purely from experience to create some level of a baseline, interviews were conducted as users were aware the prototype is a portfolio style website. Some example interview questions were:

  • Which elements do you think are clickable?
  • Did the image collage create a dizzying effect?
  • Can you assume who may need a website such as this?

    A key optimization (particularly to the mobile version) was to emphasize project cards as an app drawer from an Android or iPhone device. A user will immediately associate each case study as a potential app or similar experience. See Frame 2 (below)

    Responsive frames

    In terms of navigation, color accented vectors will help trace user's eyes towards the project-cards, respectfully. Letting the background color of black (#000000) with an opacity of 40%, creates a middle-ground of light and dark modes as an optimization of sorts. See Frame 1 (above)

    Responsive frames

    To help identify the designer as to this particular instance, a small collection of poems is available copyrighted and authored by me. Exhibiting multiple facets of the particular field may increase dynamacism to the portfolio as a whole. See Frame 3 (above)

    As a minimalistic approach to the desktop aspect, a decision was made to let this website be a single-column website with hovering effects rather than navigating to different webpages. A hover will initialize a legible iframe of a case study, however, this is only the case as to desktop; the mobile version will have thumbnails within each html page to reference the previous webpage.

    Test

    A moderated usability study was made with given tasks and prompts conjuncting with a ‘think-aloud protocol’.

    An executive decision was made into the final implementation to omit javascript. One advantage of omitting javascript is that a ‘search keyword’ function (ctrl-f) is already built into most browsers.

    Ensuing are two critical findings throughout this usability study:

      1. Omission of javascript

    An initial finding was that an effective portfolio/website can go quite far with only html/css as a static website. One can omit optional javascript to ensure speed of the website.

      2. Opt a single-column website

    A user opening html pages in new tabs while navigating may distract or create an arbitrary time-lapse, even if only an illusion. Applicable only to desktop

    Takeaways

    A design impact may be the emphasis to push html/css to the limits before the thought of javascript. Some users are aware to the fact that javascript may collect personal data arbitrarily and may create security issues, i.e. what Richard Stallman would identify as ‘the javascript trap’. A theme throughout this design seemingly is how to triangulate tiny optimizations into a high-exchanging proffer.

    UI style guide

    A primary takeaway from this design, is the idea to encapsulate a dynamicism of the user experience design field into a portfolio; as to fitting as many UX aspects into the portfolio as possible, aspects being the many subsets of UX: engineering, interaction, researching, writing et cetera.
