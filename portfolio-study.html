Case Study

A User Experience for a Portfolio & Website

Portfolio UX/UI seeks minute optimizations

By FRANCIS CACCAVALE

A design seeking optimizations to a UX portfolio/website in conjunction with responsive design while attributing to some identity that a user can relate to.

A primary target in this design was to seek small yet poignant optimizations, that will help a relatively trivial website/portfolio convey information at lightning speed yet retain aesthetics.

Understanding the user as to narrowing optimizations

A portfolio/website user-interface and user-experience has seen numerous iterations, maybe tens of thousands. A new threshhold begins as to whether an individual wants to generate a generic Squarespace et cetera templates or try a hand at html/css into the influence of Squarespace templates. The only issue is that after Squarespace generates hundreds of portfolio templates for individuals globally, the landscape may become complacent as to 'this is what a portfolio has to be, UI, fonts, colors et cetera'. This design intends to create a stochastic approach to break a mold, which means to create a baseline prototype solely from experience and build a design process from a prototype.

After creating an initial low-fidelity prototype based purely from experience to create some level of a baseline, interviews were conducted as users were aware the prototype is a portfolio style website. Some example interview questions were:

  • Which elements do you think are clickable?
  • Did the image collage create a dizzying effect?
  • Can you assume who may need a website such as this?

    Starting the design

    Beginning the design (after the baseline phase) means addressing each critical insight.

    A key optimization (particularly to the mobile version) was to emphasize project cards as an app drawer from an Android or iPhone device. A user will immediately associate each case study as a potential app or similar experience.

    Responsive display - single frame

    To help identify the designer, a small collection of poems is available authored and copyrighted by me, which pairs really well with coffee (as research indicates a portfolio/website is highly likely to be seen in a mobile device because of Twitter and such). Exhibiting multiple facets of the particular field may increase dynamacism to the portfolio as a whole.

    As a minimalistic approach to the desktop aspect, a decision was made to let this website be a single-column website with hovering effects rather than navigating to different webpages. A hover will initialize a legible iframe of a case study, however, this is only the case as to desktop; the mobile version will have thumbnails within each html page to reference the previous webpage.

    In terms of navigation, color accented vectors will help trace user's eyes towards the project-cards, respectfully.

    Refining the design as to qualitative usability testing

    A moderated usability study was made with given tasks and prompts conjuncting with a ‘think-aloud protocol’.

    An executive decision was made into the final implementation to omit javascript. One advantage of omitting javascript is that a ‘search keyword’ function (ctrl-f) is already built into most browsers.

    Ensuing are two critical findings throughout this usability study:

      1. Omission of javascript

    An initial finding was that an effective portfolio/website can go quite far with only html/css as a static website. One can omit optional javascript to ensure speed of the website.

      2. Opt a single-column website

    A user opening html pages in new tabs while navigating may distract or create an arbitrary time-lapse, even if only an illusion.

    Takeaways

    A design impact may be the emphasis to push html/css to the limits before the thought of javascript. Some users are aware to the fact that javascript may collect personal data arbitrarily and may create security issues, i.e. what Richard Stallman would identify as ‘the javascript trap’. A theme throughout this design seemingly is how to triangulate tiny optimizations into a high-exchanging proffer.

    UI style guide

    A primary consequence to this design is an implementation with html/css, in order to see just how far html/css can take this design without having to implement javascript.
